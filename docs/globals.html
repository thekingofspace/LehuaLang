Globals

A few helpers are always in scope, in the main VM and in every parallel worker, with no include needed. The library globals like fs and task have their own pages; this page collects the small standalone globals that do not belong to a single library.

GetType

GetType(value) returns the type of a value, but for a table that carries a __type marker it returns that marker instead. This is how a class reports its own type. The marker can sit on the table directly or on its metatable, and it may be a string or a function that receives the value and returns a string. Anything that is not a table, and any table without a marker, returns the normal type.

local Vec = NewClassData({
    Private = { x = 0, y = 0 },
    Public = {},
    __type = "Vector",
    __construct = function(self, x, y) self.x, self.y = x, y end,
})

print(GetType(BuildClassData(Vec, 1, 2))) -- Vector
print(GetType(Vec))                       -- ClassData
print(GetType(42))                        -- number
print(GetType("hi"))                      -- string
Why not typeof

The built-in typeof is left untouched so the type checker keeps narrowing types correctly. GetType is a separate function that returns a plain string, so comparing its result against your own type names, like GetType(v) == "Vector", does not upset the analyzer. See Metamethods for the __type marker.

print

print writes its arguments to standard output, separated by spaces and followed by a newline. It pretty-prints tables over multiple lines with sorted keys, follows nested tables, and adds color when the output is a terminal. A value with a __toconsole is shown using that instead. Strings at the top level are written as they are; inside a table they are quoted.

print("count", 3, { a = 1, b = { 2, 3 } })
-- count 3 {
--     a = 1,
--     b = {
--         2,
--         3,
--     },
-- }

To control a value's printed form, give it a __toconsole. That and the other custom markers are covered in Metamethods.

__dirname and __filename

Every module receives __dirname and __filename as the absolute folder and file path of that module on disk. They are handy for building paths or for logging where code is running from.

print(__filename) -- the full path to this file
print(__dirname)  -- the folder that holds it
