Library Reference

The cache library

Fast in-process caches with cache. Mem caches live once per program and are visible from every parallel thread, with per-entry time to live, atomic counters, and rich value support. Local LRU caches and memoization round it out.

Including cache

--#include[cache]

Mem caches

A mem cache is a named key-value store that exists once for the whole program. Every thread that opens the same name gets the same data: a value set on one thread is immediately readable from all the others, and an update made anywhere is what everyone sees next. Values are copied in and out, never shared, so there is nothing to lock in your own code.

FunctionReturnsDescription
cache.mem(name?, options?)MemCacheOpens the mem cache called name, creating it if needed. Without a name you get a fresh anonymous cache. options.ttl sets a default time to live in seconds for new entries.
cache.list(){ string }Names of every mem cache in the program.
cache.drop(name)booleanEmpties the named cache and removes it from the registry. Handles that are still open keep working but become detached from future cache.mem(name) calls.
src/main.luau
--#include[cache]

local sessions = cache.mem("sessions", { ttl = 60 })

sessions:set("user:1", { name = "Kai", level = 3 })
print(sessions:get("user:1").name) -- Kai
print(sessions:ttl("user:1")) -- just under 60

sessions:set("token", "abc123", 5) -- overrides the default ttl
sessions:increment("visits") -- atomic, starts at 0

The MemCache object

MethodReturnsDescription
c:set(key, value, ttl?)()Stores a copy of value. ttl is seconds until the entry expires; leave it out to use the cache default, or pass 0 to keep the entry forever.
c:get(key)anyA fresh copy of the stored value, or nil if missing or expired. Yields only when a value needs to reconnect, like a mongo handle.
c:take(key)anyLike get but also removes the entry.
c:add(key, value, ttl?)booleanStores only when the key is absent or expired. True when it stored. This is the building block for cross-thread locks and once-only work.
c:increment(key, delta?, ttl?)numberAtomically adds delta (default 1) to a numeric entry, creating it at 0 first. Returns the new value.
c:has(key)booleanTrue when the key holds a live entry.
c:delete(key)booleanRemoves the entry. True when a live entry was removed.
c:ttl(key)number?Seconds left before the entry expires, math.huge for entries that never expire, or nil when missing.
c:touch(key, ttl?)booleanResets the entry's time to live without changing its value.
c:keys(){ string }Sorted list of live keys.
c:count()numberNumber of live entries. #c works too.
c:clear()()Removes everything.
c:name()stringThe cache's registry name.
c:stats()table{ hits, misses, count } for this cache since the program started.

Sharing across parallel threads

Mem caches are the shared memory of Lehua's threading story. Open the same name on both sides, or simply pass the handle itself to parallel(...), through a Port, or in a messenger publish - a mem cache handle is one of the few objects that crosses threads.

src/main.luau
--#include[cache, task]

local stats = cache.mem("stats")
parallel("@self/worker.luau", stats)

task.wait(0.2)
print(stats:get("ready")) -- true, set by the worker
src/worker.luau
local stats = ...

stats:set("ready", true)
stats:increment("workers")

Rich values

Ports and parallel(...) arguments only carry simple values. Mem caches accept much more, and this is the safe way to move structured objects between threads: values are broken down into plain data when stored and rebuilt when read. A table keeps its metatable. Buffers and vectors round-trip. These objects rebuild too:

ValueHow it rebuilds
datetime objectsFrom their timestamp, exact to the microsecond.
regex objectsRecompiled from the pattern and flags.
sqlite databasesReopened from the same file. In-memory and closed databases cannot cross.
mongo clusters, databases, collections, object idsReconnected with the original connection string, once per thread.
canvas imagesRebuilt pixel by pixel.
MemCache handlesReopened by name.
What stays out

Functions, coroutines, file handles, network objects, and anything that holds a callback cannot be stored in a mem cache. They are bound to the thread that made them. Storing one raises a clear error naming the offending key.

Local LRU caches

An LRU cache lives on one thread only and can hold any Luau value, functions included. When it grows past its capacity the least recently used entry is evicted. Entries can also expire with a ttl.

FunctionReturnsDescription
cache.lru(capacity, options?)LruCacheA new local cache holding at most capacity entries. options.ttl sets a default time to live.

The object has set, get, peek (a get that does not count as a use), has, delete, keys, count, capacity, and clear.

src/main.luau
--#include[cache]

local recent = cache.lru(2)
recent:set("a", 1)
recent:set("b", 2)
recent:get("a") -- marks a as recently used
recent:set("c", 3) -- evicts b
print(recent:has("b")) -- false

Memoization

cache.memoize(fn, options?) wraps a function so repeated calls with the same arguments return the remembered result instead of running again. Arguments must be values a mem cache could store; results can be anything. options takes ttl and capacity (default 256).

src/main.luau
--#include[cache]

local slowSquare = cache.memoize(function(n)
    for _ = 1, 10000000 do end
    return n * n
end)

print(slowSquare(12)) -- slow
print(slowSquare(12)) -- instant

Lifetime and errors

Expired entries vanish lazily on access and a background sweeper tidies the rest about once a second, so a mem cache never serves stale data. Mem caches live for the whole program once created; use cache.drop when a named cache should go away. Storing an uncacheable value, incrementing a non-number, or passing a bad ttl raises an error with a cache: prefix that pcall catches like any other.
