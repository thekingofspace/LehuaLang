Library Reference

The serde library

The serde library turns Luau values into text and back. It speaks JSON, YAML, and TOML through one pair of functions, so reading a config file or writing an API payload is a single call.

Including serde

--#include[serde]

Formats

Every function takes the format name as its first argument. The name is not case sensitive.

NameFormatNotes
"json"JSONThe safe default for data exchange.
"yaml"YAML"yml" also works. Output is always indented.
"toml"TOMLThe top level value must be a table with string keys.

serde.formats is an array of these names, handy for showing what a program accepts.

Encoding

serde.encode(format, data, pretty?) returns the value as a string. Pass true as the third argument for an indented, human friendly layout. That matters for json and toml; YAML is always readable.

src/main.luau
--#include[serde, fs]

local config = {
    name = "my-app",
    version = 3,
    tags = { "fast", "small" },
}

print(serde.encode("json", config))
-- {"name":"my-app","tags":["fast","small"],"version":3}

fs.write("./config.toml", serde.encode("toml", config, true))

Decoding

serde.decode(format, text) parses a string and returns the value. Bad input raises a Lua error with the parser's message, so wrap the call in pcall when the text comes from outside your program.

src/main.luau
--#include[serde, fs]

local config = serde.decode("toml", fs.read("./config.toml"))
print(config.name, config.version)

local ok, data = pcall(serde.decode, "json", userInput)
if not ok then
    print("that was not valid JSON")
end

Null and empty tables

Luau has no null value and cannot tell an empty array from an empty map, so two helpers fill the gap.

HelperMeaning
serde.nullEncodes as null. When decoding, nulls simply become nil, which means the key disappears from the table.
serde.array(t?)Marks a table as an array so an empty one encodes as [] instead of {}. With no argument it returns a fresh empty array. Tables that already have items sort this out on their own.
src/main.luau
print(serde.encode("json", { items = serde.array(), note = serde.null }))
-- {"items":[],"note":null}

What converts and what does not

Strings, numbers, booleans, and tables of those all convert. A table with numbered items starting at 1 becomes an array — and any named keys sitting in the same table are dropped, so keep list-style and map-style data in separate tables. A table with only named keys becomes a map. Functions, userdata, tables that contain themselves, and tables nested more than 128 levels deep cannot be encoded and raise an error. TOML date values decode as plain strings.
