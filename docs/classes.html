Classes

Lehua has a small class system built into the runtime. Every module can use it with no include and no import. You describe a class with NewClassData, then stamp out objects from it with BuildClassData. A class separates private state from a public surface, holds static members that live on the class itself, supports single and multiple inheritance, checks interfaces, and lets you customize typeof and how an object prints.

The keywords

Six globals make up the system: NewClassData, BuildClassData, SuperGet, Interface, and Implements, plus a typeof that understands a __type field. They are always in scope, in the main VM and in every parallel worker.

Defining a class

NewClassData takes one table that describes the class and returns a ClassData handle. The three field buckets are Private, Public, and Static. A __construct function runs when an object is built, and any __ key becomes a metamethod on the built objects.

src/counter.luau
local Counter = NewClassData({
    Name = "Counter",
    Private = {
        count = 0,
    },
    Public = {
        Increment = function(self, by)
            self.count = self.count + (by or 1)
            return self.count
        end,
        Get = function(self)
            return self.count
        end,
    },
    Static = {
        Version = "1.0",
    },
    __construct = function(self, start)
        if start ~= nil then
            self.count = start
        end
    end,
})

The table may hold these keys:

KeyMeaning
PrivateFields that only methods can reach, through self. Each object gets its own copy.
PublicMethods and fields callers may use. Each object gets its own copy.
StaticMembers that live on the class itself, shared by every object.
__constructRuns during BuildClassData as __construct(self, ...) with the build arguments.
InheritsA parent class, or a list of parents. See Inheritance.
InterfacesA list of interfaces the class must satisfy. Checked when the class is created.
NameA label used in the default console output.
__add, __tostring, __type, __toconsole, ...Any __ key other than __construct becomes a metamethod on built objects.

Building objects

BuildClassData takes a class and any constructor arguments, and returns a fresh object. It deep-copies the Private and Public tables into the new object, so one object never shares state with another. Functions are shared by reference, but data, including nested tables, is copied.

local a = BuildClassData(Counter, 10)
local b = BuildClassData(Counter)

print(a:Increment(5)) -- 15
print(b:Get())        -- 0, b is untouched by changes to a

Private, public, and self

Inside a method, self reaches everything the object holds, both public and private. From outside, only public members are visible. Reading or writing a private field from outside raises a Lua error, so private really means private.

local c = BuildClassData(Counter)

print(c:Get())          -- 0, a public method

local ok = pcall(function()
    return c.count      -- error: cannot read private field 'count'
end)
print(ok)               -- false
Note

A method receives the object as self, so self.count reads the private field and self:Get() calls another public method. External code goes through the object's public surface, where private keys are hidden and assigning to one is an error.

Statics

Static members live on the class, not on any object. You reach them straight from the class handle, and also from any object built from it.

print(Counter.Version)  -- 1.0
local c = BuildClassData(Counter)
print(c.Version)        -- 1.0

Counter.Version = "2.0"
print(c.Version)        -- 2.0, statics are shared

When a class inherits a static, that static keeps pointing back to the class that defined it. Reading or writing an inherited static goes to the origin class, so there is one shared value, not a copy per child.

typeof and __type

Lehua's typeof works exactly like the built-in for every value except a table that carries a __type marker. The marker can sit on the table directly or on its metatable, and it may be a string or a function that returns a string. Anything that is not a table, and any table without a marker, falls through to the normal typeof.

local Vec = NewClassData({
    Private = { x = 0, y = 0 },
    Public = {},
    __type = "Vector",
    __construct = function(self, x, y)
        self.x, self.y = x, y
    end,
})

print(typeof(BuildClassData(Vec, 1, 2))) -- Vector
print(typeof(Vec))                       -- ClassData
print(typeof(42))                        -- number
print(typeof("hi"))                      -- string

A function marker receives the object, so the type can depend on the object's state.

local Tagged = NewClassData({
    Private = { tag = "?" },
    Public = {},
    __type = function(self)
        return "Tagged:" .. self.tag
    end,
    __construct = function(self, tag)
        self.tag = tag
    end,
})

print(typeof(BuildClassData(Tagged, "widget"))) -- Tagged:widget

Printing with __toconsole

Give a class a __toconsole and print uses it to show the object. It can be a string, or a function that receives the object and returns text. Without one, an object prints as its class name followed by its public data.

local Vec = NewClassData({
    Name = "Vec",
    Private = { x = 0, y = 0 },
    Public = {},
    __toconsole = function(self)
        return "Vector(" .. self.x .. ", " .. self.y .. ")"
    end,
    __construct = function(self, x, y)
        self.x, self.y = x, y
    end,
})

print(BuildClassData(Vec, 1, 2)) -- Vector(1, 2)

Metamethods

Any __ key other than __construct becomes a metamethod on built objects. Operators like __add, __eq, and __tostring all work, and each object argument arrives as its own self, so a metamethod can read private state.

local Vec
Vec = NewClassData({
    Private = { x = 0, y = 0 },
    Public = {
        Sum = function(self) return self.x + self.y end,
    },
    __add = function(self, other)
        return BuildClassData(Vec, self.x + other.x, self.y + other.y)
    end,
    __construct = function(self, x, y)
        self.x, self.y = x, y
    end,
})

local sum = BuildClassData(Vec, 1, 2) + BuildClassData(Vec, 3, 4)
print(sum:Sum()) -- 10
Referring to a class from inside itself

A metamethod that builds another object of the same class needs the class in scope. Declare the local first, then assign it, as in local Vec followed by Vec = NewClassData({ ... }). Otherwise the name is not yet visible inside the table.

Inheritance

Set Inherits to a parent class to take everything it defines. The child's own Public, Private, and Static override anything with the same name from the parent.

local Animal = NewClassData({
    Private = { name = "?" },
    Public = {
        Speak = function(self) return self.name .. " makes a sound" end,
        GetName = function(self) return self.name end,
    },
    __construct = function(self, name) self.name = name end,
})

local Dog = NewClassData({
    Inherits = Animal,
    Public = {
        Speak = function(self) return self.name .. " barks" end,
    },
    __construct = function(self, name) self.name = name end,
})

local d = BuildClassData(Dog, "Rex")
print(d:Speak())   -- Rex barks, overridden
print(d:GetName()) -- Rex, inherited

Multiple parents

To inherit from more than one parent, list the parents and name which keys to take from each, split by kind. This keeps inheritance explicit, so you pick exactly the fields you want and reach the rest with SuperGet.

local Widget = NewClassData({
    Name = "Widget",
    Inherits = {
        { Class = Disposable, Public = { "Destroy" }, Private = { "disposed" } },
        { Class = Named, Public = { "GetLabel", "SetLabel" }, Private = { "label" }, Static = { "Registry" } },
    },
    Private = { width = 0 },
    Public = {
        GetWidth = function(self) return self.width end,
    },
    __construct = function(self, label, width)
        self.label, self.width = label, width
    end,
})

Each entry is a table with a Class and lists of key names under Public, Private, Static, and Metamethods. Only the listed keys are inherited directly. A bare class in the list, or Inherits = Parent on its own, takes everything from that parent.

Name clashes are an error

Inheriting the same key from two parents raises an error, and so does naming a key the parent does not have. When two parents share a name, take it from only one and reach the other through SuperGet.

SuperGet

SuperGet pulls a raw field or method off a class without binding it. You then call it yourself and pass the object as the first argument. This reaches a parent's members whether or not the child inherited them directly, which is how you call a base method after overriding it.

local Dog = NewClassData({
    Inherits = Animal,
    Public = {
        Speak = function(self) return self.name .. " barks" end,
        SpeakLikeAnimal = function(self)
            return SuperGet(Animal, "Speak")(self)
        end,
    },
    __construct = function(self, name) self.name = name end,
})

local d = BuildClassData(Dog, "Rex")
print(d:SpeakLikeAnimal()) -- Rex makes a sound

SuperGet looks through the class's public, then private, then static, then metamethod members, and returns the first match. Since it hands back the raw function, call it from inside a method where self is the full object, so the function can reach private fields.

Interfaces

Interface describes a set of members a class must provide. List the members as a table, or pass a table with a Name and a Requires list. A class that names the interface under Interfaces is checked the moment it is created, and fails loudly if a required member is missing.

local IShape = Interface({ Name = "IShape", Requires = { "Area" } })

local Circle = NewClassData({
    Private = { r = 0 },
    Public = {
        Area = function(self) return 3.14 * self.r * self.r end,
    },
    Interfaces = { IShape },
    __construct = function(self, r) self.r = r end,
})

local c = BuildClassData(Circle, 2)
print(Implements(c, IShape))            -- true
print(Implements(c, Interface({ "X" }))) -- false

Implements(value, interface) returns whether an object or a class provides every member the interface requires. It checks the public surface and statics.

Errors

Misusing the class functions raises Lua errors: reading or writing a private field from outside, passing a non-class to BuildClassData or SuperGet, clashing inherited names, or missing an interface member. Wrap a call in pcall when you want to handle these yourself.
