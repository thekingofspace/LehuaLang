Parallel

The parallel keyword runs a file on its own operating system thread with its own Luau VM. This is real parallelism, not cooperative coroutines. It returns a Port, which is the parent end of a two way channel. Inside the worker file, the matching end is the global channel. The worker path resolves like any relative path: at a module's top level it is the module's own folder (inside an init.luau, the folder's parent), and inside a function called after load the folder of the file that started the call is tried first, then each module along the chain, so a module can spawn a worker from a path its caller handed in. Use @self/worker.luau to spawn a file inside the module's own folder.

src/main.luau
local worker = parallel("./worker.luau")

for i = 1, 5 do
    worker:Push(i)
    print(worker:Pop()) -- Pop yields until the worker answers
end

worker:Close()
src/worker.luau
-- runs on its own thread
while true do
    local n = channel:Pop()
    if n == nil then
        break -- nil means the parent closed the channel
    end
    channel:Push(n * 2)
end

A Port has Push(value), Pop() which yields, TryPop() which does not yield, Close(), and IsClosed().

Note

Only simple values cross a Port. That means nil, booleans, numbers, strings, and plain tables. Values are copied, not shared. Functions, threads, and userdata cannot cross, since each worker is a separate VM. The one exception is a mem cache handle, which crosses by name.

Beyond Ports

Ports pair two threads. For broadcast-style events that any thread can hear, use the messenger global. For shared state with expiring entries, or to move richer objects like datetimes, regexes, and database handles between threads, use a mem cache.
