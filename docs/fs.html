Library Reference

The fs library

The fs library reads and writes files and folders, and helps you build and inspect paths. It works like Lua's io library, with file handles and simple whole file helpers on top.

Including fs

Add the include header to any file that uses it. Only libraries you include are set up, which keeps the built executable small.

--#include[fs]

How paths are resolved

Every function that takes a path follows the same rules. While a module's own top level code is running, a relative path stays inside the module's own folder, so a module can reach its files on initial load. Inside a function called after load, the path resolves along the call chain: the folder of the file that started the call is tried first, then each module along the chain, and the first place where the path exists wins. A path that exists nowhere yet, for example a file about to be written, lands next to the file that started the call. This means a module function can take a path from its caller and it will mean what the caller expects. Use @self when a module wants its own folder no matter who called it. An absolute path is used as it is.

FormMeaning
./namenext to the running script, then along the call chain
../nameup one folder, resolved the same way
@self/namethe current module's own folder
@Alias/namean alias from .luaurc, for example @root
/x or C:/xan absolute path, used as given
Errors

The reading and writing functions raise a Lua error when something goes wrong, such as a missing file or no permission. Wrap a call in pcall if you want to handle that yourself.

Path helpers

These work with strings only. They never touch the disk.

FunctionReturnsDescription
fs.resolve(path)stringThe full, cleaned path after applying the rules above.
fs.join(...)stringJoins the given parts into one path.
fs.dirname(path)stringThe folder part of a path.
fs.basename(path)stringThe last part of a path, the file name.
fs.extname(path)stringThe file ending with a leading dot, or an empty string.
fs.cwd()stringThe current working directory of the process.
src/main.luau
print(fs.resolve("@self/data.txt")) -- a full path
print(fs.join("a", "b", "c.txt"))   -- a/b/c.txt
print(fs.dirname("a/b/c.txt"))      -- a/b
print(fs.basename("a/b/c.txt"))     -- c.txt
print(fs.extname("a/b/c.txt"))      -- .txt

Checks

FunctionReturnsDescription
fs.exists(path)booleanTrue if a file or folder is there.
fs.isFile(path)booleanTrue if the path is a file.
fs.isDir(path)booleanTrue if the path is a folder.

Metadata

fs.metadata(path) returns a table that describes a file or folder. It raises an error if the path does not exist.

FieldTypeDescription
typestring"file", "dir", or "other".
sizenumberThe size in bytes. For folders this value is system dependent.
readonlybooleanTrue if the file cannot be written to.
modifiedAtnumber?When it was last changed, in seconds since the Unix epoch.
createdAtnumber?When it was made, in seconds since the Unix epoch.
accessedAtnumber?When it was last read, in seconds since the Unix epoch.

The time fields can be missing on file systems that do not track them. They include fractions of a second, so pass them through math.floor before handing them to os.date.

src/main.luau
local meta = fs.metadata("./notes.txt")
print(meta.type, meta.size)
if meta.modifiedAt then
    print("last changed " .. os.date("%Y-%m-%d %H:%M", math.floor(meta.modifiedAt)))
end

Reading and writing whole files

FunctionReturnsDescription
fs.read(path)stringThe whole file as a string. Works with text or binary.
fs.write(path, data)nilWrites data to the file, replacing what was there.
fs.readBuffer(path)bufferThe whole file as a Luau buffer.
fs.writeBuffer(path, data)nilWrites a Luau buffer to the file, replacing what was there.
fs.readImage(path)CanvasDecodes an image file into a canvas. Needs the canvas library.
fs.writeImage(path, canvas, format?, quality?)stringEncodes a canvas to a file; the format defaults to the extension. Returns the path.
fs.append(path, data)nilAdds data to the end of the file, making the file if needed.
fs.readDir(path){ string }An array of the names inside a folder.
src/main.luau
fs.write("./notes.txt", "first line\n")
fs.append("./notes.txt", "second line\n")
print(fs.read("./notes.txt"))

for _, name in fs.readDir("@self") do
    print(name)
end

The buffer pair is for files with custom binary layouts. Build the bytes with the buffer standard library, write them out, and read them back without any string conversion in between.

src/main.luau
local b = buffer.create(8)
buffer.writeu32(b, 0, 42)
buffer.writef32(b, 4, 1.5)
fs.writeBuffer("./save.dat", b)

local back = fs.readBuffer("./save.dat")
print(buffer.readu32(back, 0), buffer.readf32(back, 4)) -- 42, 1.5

Finding files with glob

fs.glob(pattern) returns an array of full paths that match a pattern. Where fs.readDir lists everything in one folder by name, glob filters by shape and can search whole trees. The pattern is resolved like any other path, so ./*.txt looks next to the current script. A pattern that is not valid raises an error.

WildcardMatches
*any run of characters within one name
?exactly one character
[abc]one character from the set. Ranges like [0-9] work too.
**any number of folders deep, for searching a whole tree
src/main.luau
-- every .luau file next to this script
for _, path in fs.glob("./*.luau") do
    print(path)
end

-- every .json file under data, however deep
for _, path in fs.glob("./data/**/*.json") do
    print(fs.basename(path))
end

Folders and removing

FunctionReturnsDescription
fs.mkdir(path)nilMakes a folder and any parent folders it needs.
fs.remove(path)nilRemoves a file, or an empty folder.
fs.removeAll(path)nilRemoves a file, or a folder and everything inside it.

Copying and moving

FunctionReturnsDescription
fs.copy(src, dst)nilCopies one file, replacing the destination if it is already there. Raises an error if the source is a folder.
fs.copyAll(src, dst)nilCopies a file, or a folder and everything inside it. Makes any parent folders the destination needs.
fs.move(src, dst)nilMoves a file or folder. It tries a fast rename first; when that is not possible, for example across drives, it copies everything and then deletes the original.
A folder cannot move into itself

fs.copyAll and fs.move raise an error when the destination is inside the source folder, since that copy would never end.

src/main.luau
fs.copy("./config.toml", "./config.backup.toml")
fs.copyAll("./assets", "./dist/assets")
fs.move("./old-name.txt", "./new-name.txt")

Temporary files and folders

These make a fresh entry in the operating system's temp folder and return its full path. Every call picks a new unique name, so parallel runs never clash. The entry really exists on disk when the call returns. Temp space is for short lived scratch work; remove what you made with fs.removeAll when you are done.

FunctionReturnsDescription
fs.tempDir()stringMakes a new empty folder and returns its path.
fs.tempFile(extension?)stringMakes a new empty file and returns its path. The extension defaults to tmp; a leading dot is fine.
src/main.luau
local dir = fs.tempDir()
fs.write(fs.join(dir, "work.txt"), "scratch data")

local log = fs.tempFile("log") -- ends in .log
fs.append(log, "started\n")

fs.removeAll(dir)

fs.open and the File object

fs.open(path, mode?) opens a file and returns a File object. The default mode is "r". Call close when you are done.

ModeMeaning
"r"read
"w"write, empties the file first
"a"append, writes at the end
"r+"read and write
"w+"read and write, empties the file first
"a+"read and append

Add b for binary, such as "rb". On most systems this makes no difference.

File methods

MethodReturnsDescription
file:read(format?)string | number | nilReads from the file. See the formats below. Returns nil at the end of the file.
file:write(...)nilWrites strings or numbers.
file:seek(whence?, offset?)numberMoves the read and write spot. Returns the new spot, counted from the start.
file:lines()() -> string?Returns a function for a for loop. Each call gives the next line, or nil at the end.
file:flush()nilPushes any buffered writes to disk.
file:close()nilCloses the file. Using it after this raises an error.

Read formats

FormatReturnsMeaning
"a"stringThe rest of the whole file.
"l"string?The next line without the newline. This is the default.
"L"string?The next line with the newline.
"n"number?Reads a number.
a count, like 16string?Up to that many bytes.

seek whence values

ValueMeaning
"set"from the start of the file
"cur"from the current spot, the default
"end"from the end of the file
src/main.luau
local file = fs.open("./notes.txt", "r")
for line in file:lines() do
    print(line)
end
file:seek("set", 0)     -- back to the start
print(file:read("a"))   -- the whole file
file:close()
