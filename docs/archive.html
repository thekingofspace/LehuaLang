Library Reference

The archive library

The archive library bundles files into zip and tar archives and unpacks them again. An archive is a single file that holds many files and folders, which makes backups and downloads easy to move around.

Including archive

--#include[archive]

Sources and paths

Every function that creates an archive takes a sources argument. It can be one path or an array of paths. Each source is added to the archive under its own name, so "./build/output" shows up inside as output. A folder is added recursively: the folder itself and everything inside it, under the folder's name.

Paths follow the same rules as the fs library. A relative path resolves against the folder of the script that included archive, an absolute path is used as it is, and @self or a .luaurc alias works too. A source that does not exist, an empty array, or a value that is not a path raises a Lua error before anything is written.

Zip archives

Zip is the common choice: it compresses every file and opens everywhere, including Windows Explorer.

FunctionReturnsWhat it does
archive.zip(dest, sources, options?)nothingCreates a zip file at dest containing the sources. options is a table; options.level sets the compression level.
archive.unzip(src, destFolder)nothingExtracts every entry of the zip into destFolder, creating the folder if it is missing.
archive.listZip(src)arrayReturns the entries without extracting anything.

options.level ranges from 0 to 9. Higher levels make smaller files but take longer; 0 stores the data with almost no compression. Leave the option out for a balanced default.

Each entry from archive.listZip is a table with these fields:

FieldTypeMeaning
namestringThe path inside the archive, with forward slashes.
sizenumberThe file's size in bytes before compression.
compressedSizenumberThe size in bytes inside the archive.
isDirbooleantrue when the entry is a folder rather than a file.
src/main.luau
--#include[archive]

-- one file and one folder, compressed as small as possible
archive.zip("./backup.zip", { "./notes.txt", "./photos" }, { level = 9 })

for _, entry in archive.listZip("./backup.zip") do
    print(entry.name, entry.size, "->", entry.compressedSize)
end

archive.unzip("./backup.zip", "./restored")
-- ./restored/notes.txt and ./restored/photos/... now exist

Tar archives

Tar is the usual format on Linux and macOS. A plain tar file only bundles files without shrinking them, so it is almost always paired with gzip compression.

FunctionReturnsWhat it does
archive.tar(dest, sources)nothingCreates a tar file at dest. When dest ends in .gz or .tgz, the result is gzip compressed as well.
archive.untar(src, destFolder)nothingExtracts the tar into destFolder, creating the folder if it is missing. Gzip is detected from the file's contents, so the name does not matter.
archive.listTar(src)arrayReturns the entries without extracting anything. Detects gzip the same way.

Entries from archive.listTar have the same name, size, and isDir fields as zip entries. There is no compressedSize, because gzip compresses the whole archive at once rather than each file.

src/main.luau
--#include[archive]

-- the .gz ending turns on gzip compression
archive.tar("./site-backup.tar.gz", "./site")

for _, entry in archive.listTar("./site-backup.tar.gz") do
    print(entry.name, entry.isDir)
end

archive.untar("./site-backup.tar.gz", "./restored")
-- ./restored/site/... now exists
Extraction is sandboxed to the destination

A malicious archive can contain entry names like ../../secret that try to write outside the folder you extract into. archive.unzip and archive.untar refuse such paths, so extraction never touches anything outside the destination folder.

Errors

Problems raise a Lua error: a source that does not exist, a destination that cannot be written, a corrupt archive, or a source path with no usable name such as a bare drive root. Wrap calls in pcall when the archive comes from outside your program.
