Library Reference

The stdio library

The stdio library is how a program talks with the terminal. It prints text exactly as you give it, reads what the user types, and colors output with ANSI escape codes.

Including stdio

--#include[stdio]

Writing

stdio.write(...) prints its arguments to standard output. stdio.ewrite(...) does the same on standard error, the stream terminals set aside for warnings and error messages. Both accept any mix of strings, numbers, and booleans, join them with nothing in between, and add no newline at the end. Anything else, like a table or nil, raises a Lua error. stdio.writeLine(...) and stdio.ewriteLine(...) are the same but finish with a newline.

Compare with print, which puts a space between values and a newline at the end. With write you place every character yourself, which is exactly what you want for progress output and colored text.

Two helpers tidy the screen: stdio.clear() wipes the whole terminal and moves the cursor to the top left, and stdio.clearLine() erases the current line and returns the cursor to its start, which is handy for progress counters that redraw in place.

How print shows values

Lehua's print is made for looking at data. Tables are printed with their whole content: list items first, then named keys in alphabetical order, indented one level per depth. Nested tables print nested; tables that contain themselves and tables deeper than six levels print as {...} instead of running forever. A table with a __tostring metamethod prints whatever that returns instead of its content. Library objects like a DateTime, a Regex, or a MongoCluster print as their name in square brackets, in bold blue when the output is a real terminal: [MongoCluster]. In a terminal the rest is colored by type too: keys cyan, numbers yellow, booleans blue, nil gray, functions magenta. When output is piped to a file the colors are left out automatically.

src/main.luau
--#include[datetime]

print({ 3, 1, name = "ada", nested = { on = true } })
-- {
--     3,
--     1,
--     name = "ada",
--     nested = {
--         on = true,
--     },
-- }

print(datetime.now())  -- [2026-07-08T19:04:05.123Z]
src/main.luau
--#include[stdio]

stdio.write("2 + 2 = ", 4, "\n")
stdio.write("finished: ", true, "\n")
stdio.ewrite("warning: running low on disk space\n")

Reading input

Three functions read from standard input, which is whatever the user types or whatever another program pipes in.

FunctionReturns
stdio.readLine()The next line as a string, without the trailing newline. Returns nil when the input ends.
stdio.readAll()Everything that is left, as one string. An empty string if there is nothing.
stdio.prompt(message?, options?)Prints the message, then reads an answer. With an options list it shows a numbered menu instead; see below. Returns nil when the input ends.

Because readLine returns nil only at the end of input, you can loop until it does. An empty line comes back as "", not nil.

src/main.luau
--#include[stdio]

-- Count the lines piped in, for example:
--   lehua run src/main.luau < notes.txt
local count = 0
local line = stdio.readLine()
while line ~= nil do
    count += 1
    line = stdio.readLine()
end
stdio.write("read ", count, " lines\n")
Reading waits

All three functions block until input arrives. If nothing is typed and nothing is piped in, the program sits and waits. To end the input by hand, press Ctrl+Z then Enter on Windows, or Ctrl+D elsewhere.

Asking a question

stdio.prompt rolls the two common steps into one call: show a message, wait for an answer.

src/main.luau
--#include[stdio]

local name = stdio.prompt("What is your name? ")

if name == nil then
    stdio.ewrite("input ended before an answer\n")
elseif name == "" then
    stdio.write("The quiet type. Nice to meet you anyway.\n")
else
    stdio.write("Hello, ", name, "!\n")
end

Choosing from a list

Pass an array of choices as the second argument and prompt turns into a small menu. In a terminal the menu is interactive: the arrow keys (or j/k, or a digit) move the highlight and Enter confirms. Esc or Ctrl+C cancels. When input or output is redirected it falls back to printing numbered choices and a > marker, then keeps asking until the answer is a valid number or matches one of the choices by name (capitalization does not matter). It returns two values: the chosen text and its number. Both are nil if the selection is cancelled or the input ends first. An empty options list raises an error.

src/main.luau
--#include[stdio]

local choice, index = stdio.prompt("Pick a color:", { "red", "green", "blue" })
-- Pick a color:
-- > red
--   green
--   blue

if choice then
    stdio.write("you picked ", choice, " (option ", index, ")\n")
end

Checking for a terminal

stdio.isTTY(stream?) returns true when a stream is connected to a real terminal and false when it is redirected to a file or another program. The stream is "stdin", "stdout", or "stderr"; leaving it out means "stdout". Any other name raises a Lua error.

Use it to decide whether colors and prompts make sense. A program whose output is piped into a file should usually skip both.

Colors and styles

stdio.color(name), stdio.bgColor(name), and stdio.style(name) each return a short string called an ANSI escape code. The string itself is invisible. When a terminal prints it, everything printed after it changes appearance, until a reset code switches things back. So coloring text is just string concatenation.

color sets the text color and bgColor sets the background behind it. Both accept the same names:

GroupNames
Normalblack, red, green, yellow, blue, magenta, cyan, white
BrightbrightBlack, brightRed, brightGreen, brightYellow, brightBlue, brightMagenta, brightCyan, brightWhite
HexAny hex code like #ff8800 or the short form #f80 picks an exact color on terminals with truecolor support.
Specialreset switches back to the default color: the text color in color, the background in bgColor.

stdio.rgb(r, g, b) and stdio.bgRgb(r, g, b) do the same as a hex code but take the three channels as numbers from 0 to 255.

style changes how the text is drawn rather than its color:

NameEffect
resetTurns off all colors and styles at once - the usual way to end colored output.
boldHeavier text.
dimFainter text.
italicSlanted text.
underlineA line under the text.
blinkFlashing text. Many terminals ignore it, mercifully.
reverseSwaps the text and background colors.
strikethroughA line through the text.

Passing a name that is not in these tables raises a Lua error that lists the supported names.

src/main.luau
--#include[stdio]

local reset = stdio.style("reset")

if stdio.isTTY() then
    stdio.write(stdio.color("green"), "ok", reset, " all tests passed\n")
    stdio.write(stdio.style("bold"), stdio.color("red"), "error:", reset, " file not found\n")
    stdio.write(stdio.bgColor("blue"), stdio.color("brightWhite"), " INFO ", reset, " build finished\n")
else
    stdio.write("ok all tests passed\n")
end
Where colors work

Escape codes render in modern terminals: Windows Terminal, the macOS Terminal, and just about every Linux terminal. When output goes to a file or another program, the codes are written out as literal characters instead. Check stdio.isTTY() before coloring, or clean the text up afterwards with stripColors.

Transcribing prints

stdio.transcribe(fn) runs the function you pass it, and while it runs, every print call is cached instead of written. When the function returns, everything it printed comes out together as one line. That makes it easy to build a line piece by piece, mixing colors and styles programmatically, without juggling one giant concatenation yourself. The combined line is also returned as a string.

src/main.luau
--#include[stdio]

local line = stdio.transcribe(function()
    print(stdio.color("brightBlack") .. "[" .. os.date("!%Y-%m-%dT%H:%M:%SZ") .. "]" .. stdio.color("reset"))
    print(" " .. stdio.color("cyan") .. "[EVENT]" .. stdio.color("reset"))
    print(" - Event named start fired")
end)
-- [2026-07-08T19:04:05Z] [EVENT] - Event named start fired

Nothing is printed if the function raises an error; the error travels on as usual. Nested transcribe calls fold their output into the surrounding one.

Stripping colors

stdio.stripColors(text) returns the text with every ANSI escape sequence removed. Handy for writing colored output to a log file, or for measuring how wide a string will actually look on screen.

src/main.luau
local fancy = stdio.color("red") .. "alert" .. stdio.style("reset")
print(#fancy)                      -- 14
print(#stdio.stripColors(fancy))   -- 5
