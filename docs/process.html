Library Reference

The process library

The process library runs other programs, reads and sets environment values, reports facts about the machine, and lets you hook into the program's own lifecycle with a heartbeat and a close handler.

Including process

--#include[process]
Threads

Running a program with run or exec, and waiting with sleep, happen on a background thread. They pause only the coroutine that called them, so the rest of your program and any parallel workers keep going.

Fields

FieldTypeDescription
process.args{ string }The arguments passed after -- on the command line.
process.pidnumberThe id of this process.
process.platformstringThe operating system, such as "windows", "linux", or "macos".
process.archstringThe cpu type, such as "x86_64" or "aarch64".

Environment variables

FunctionReturnsDescription
process.getenv(name)string?The value, or nil if it is not set.
process.setenv(name, value?)nilSets the value. Pass nil to remove it.
process.env(){ [string]: string }A table of every environment variable.

Running programs

process.run starts a program directly, with no shell. You pass the program and a list of arguments. process.exec runs a command line through the system shell, so pipes and other shell features work. Both wait for the program to finish and return a RunResult.

FunctionReturnsDescription
process.run(program, args?, opts?)RunResultRuns a program with a list of arguments.
process.exec(command, opts?)RunResultRuns a command line through the shell.

The optional opts table can set the working directory:

OptionTypeDescription
opts.cwdstring?Run in this folder. Accepts the same path forms as fs.

The RunResult object

FieldTypeDescription
okbooleanTrue if the program finished with code 0.
codenumberThe exit code. This is -1 if the program was stopped by a signal.
stdoutstringEverything the program wrote to standard output.
stderrstringEverything the program wrote to standard error.
src/main.luau
local r = process.run("git", { "status", "-s" }, { cwd = "@root" })
if r.ok then
    print(r.stdout)
else
    print("git failed with code", r.code, r.stderr)
end

local out = process.exec("echo hello | sort")
print(out.stdout)

Timing and lifecycle

FunctionReturnsDescription
process.sleep(ms)nilWaits for the given milliseconds. Yields, so other coroutines keep running.
process.exit(code?)never returnsRuns the close handlers, then ends the program with the given code. The default code is 0.

BindToHeartbeat

A heartbeat callback runs about 60 times a second, once your main code returns. It receives dt, the number of seconds since the last beat. This is handy for a game loop or any steady update.

Each binding has a name. Binding the same name again replaces its callback. The heartbeat keeps the program alive, so it runs until you remove every binding or call process.exit.

FunctionReturnsDescription
process.BindToHeartbeat(name, callback)nilBinds a callback under a name. It is called as callback(dt).
process.UnbindFromHeartbeat(name)booleanRemoves the binding with that name. Returns true if one was there.
src/main.luau
--#include[process]

local elapsed = 0
process.BindToHeartbeat("tick", function(dt)
    elapsed += dt
    if elapsed >= 3 then
        print("three seconds passed, stopping")
        process.UnbindFromHeartbeat("tick")
    end
end)

print("main returned, the heartbeat now drives the program")
Note

The heartbeat only runs after your main chunk returns. If your main code has its own while true loop that never returns, the heartbeat never starts.

BindToClose

A close callback runs when the program is about to end, whether that is a normal finish, an error, or a call to process.exit. It receives the exit code as a number.

Close callbacks run one after another, synchronously, so keep them quick. Writing a file with fs is fine. Avoid calls that yield inside a close callback, such as process.run, process.exec, process.sleep, or a Port Pop.

FunctionReturnsDescription
process.BindToClose(callback)nilAdds a callback. It is called as callback(code). You can add more than one.
src/main.luau
--#include[process]

process.BindToClose(function(code)
    print("shutting down, exit code:", code)
end)

Machine information

FunctionReturnsDescription
process.cpus()numberHow many cpu threads are available.

The Memory object

FieldTypeDescription
totalnumberTotal physical memory in bytes.
availablenumberMemory that can still be used, in bytes.
usednumberMemory in use across the machine, in bytes.
processUsednumber?Memory this process is using, in bytes.

The System object

FieldTypeDescription
osstringThe operating system name, like "windows".
archstringThe cpu type.
familystringThe family, like "windows" or "unix".
namestringThe friendly system name.
kernelstringThe kernel version.
versionstringThe long system version.
hostnamestringThe machine's host name.
src/main.luau
local mem = process.memory()
print("total MB:", math.floor(mem.total / 1048576))
print("this process MB:", math.floor((mem.processUsed or 0) / 1048576))

local sys = process.system()
print(sys.name, sys.kernel, "on", sys.hostname)
print("cpu threads:", process.cpus())
