Library Reference

The task library

The task library runs code concurrently on Lehua's cooperative scheduler. Spawn work now, defer it to the end of the cycle, delay it, or repeat it on a timer. Everything runs in its own coroutine on the same thread, so there are no locks to think about, and pending tasks keep the program alive until they finish or are cancelled.

Including task

--#include[task]

Functions

Every spawning function takes a function or an existing coroutine, followed by arguments to pass to it, and returns a Task object. Callbacks always run in their own coroutine, so they can yield with task.wait or call async things without blocking anything else.

FunctionReturnsDescription
task.wait(seconds?)numberYields the current coroutine for at least seconds, or until the next scheduler cycle when left out. Returns the time actually waited.
task.spawn(callback, ...)TaskRuns the callback in a new coroutine right away.
task.defer(callback, ...)TaskRuns the callback at the end of the current scheduler cycle, after everything already queued has had a turn.
task.delay(seconds, callback, ...)TaskRuns the callback once after seconds have passed.
task.schedule(callback, seconds, ...)TaskRuns the callback every seconds, forever, until cancelled.
task.reschedule(task, seconds)nilChanges a delay's remaining time or a schedule's repeat interval. The timer restarts from now with the new value.
task.cancel(target)nilCancels a Task or a plain coroutine: a run that has not started is prevented, a suspended run is stopped, and a schedule is unscheduled.
src/main.luau
--#include[task]

task.spawn(function(name)
    for i = 1, 3 do
        print(name .. " tick " .. i)
        task.wait(1)
    end
end, "worker")

task.delay(2, function()
    print("two seconds later")
end)

local beat = task.schedule(function()
    print("heartbeat")
end, 0.5)

task.delay(5, function()
    task.cancel(beat)
    print("stopped the heartbeat")
end)

Functions or coroutines

Anywhere a callback is accepted you can also pass a coroutine made with coroutine.create. For spawn, defer, and delay it is resumed once with the arguments you gave. For schedule it is resumed every interval, which lets a generator style coroutine carry state between runs with plain locals; the schedule ends on its own when the coroutine finishes.

src/main.luau
--#include[task]

local counter = coroutine.create(function()
    for i = 1, 5 do
        print("run " .. i)
        coroutine.yield()
    end
end)

task.schedule(counter, 1)

Yielding and resuming by hand

Every thread the runtime drives, whether it is the main script, module code, or a spawned task, follows the same rule: a plain coroutine.yield() suspends that thread indefinitely. It stays suspended until something resumes it, and the values passed to coroutine.resume(thread, ...) become the return values of the yield call. task.spawn(thread, ...) and task.defer(thread, ...) wake a suspended thread the same way. This is the building block for request and reply patterns, where a thread parks itself until a listener delivers an answer.

src/main.luau
--#include[task]

local main = coroutine.running()

task.spawn(function()
    task.wait(1)
    coroutine.resume(main, "wakey wakey")
end)

print(coroutine.yield()) -- prints "wakey wakey" one second later

Three details to know. coroutine.resume on a suspended runtime thread hands it back to the scheduler and returns true right away rather than running it inline. Resuming a runtime thread that is not suspended by yield returns false and a message. A suspended thread does not keep the program alive by itself: when nothing that could resume it remains, the program simply moves on, so a script whose last live thread is suspended exits cleanly. Coroutines you drive yourself with coroutine.create and coroutine.resume are untouched by all of this, and a coroutine handed to task.schedule keeps its generator behavior of one resume per interval.

The Task object

Every spawning function returns a Task, a handle you can keep to control the work later. The free functions task.cancel(t) and task.reschedule(t, s) do the same things as the methods.

TaskReturnsDescription
t:Cancel()nilCancels the task, same as task.cancel(t).
t:Reschedule(seconds)nilChanges the timer, same as task.reschedule(t, seconds).
t:IsCancelled()booleanWhether the task has been cancelled.
t:IsFinished()booleanWhether the task has finished all its work, including by cancellation.
t:Kind()stringOne of "spawn", "defer", "delay", or "schedule".

Lifetime and errors

A pending or repeating task keeps the program running after the main script ends, the same way a bound net receiver does. A task.schedule in particular runs forever, so cancel it when you are done or the program will not exit. Errors raised inside a task are printed to stderr and do not stop the rest of the program.

Cancelling yourself

A callback can cancel its own task. The cancelled flag is set right away and a schedule stops repeating, but the code that called cancel finishes its current run.
