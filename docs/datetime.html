Library Reference

The datetime library

The datetime library works with points in time as objects. You make a DateTime with one of the constructors, then call methods on it, compare two of them, or subtract one from another to get the seconds between.

Including datetime

--#include[datetime]

Making a DateTime

ConstructorDescription
datetime.now()The current moment.
datetime.fromUnix(seconds)From seconds since the Unix epoch. Fractions are kept.
datetime.fromMillis(ms)From milliseconds since the Unix epoch, the JavaScript convention.
datetime.fromISO(text)From an ISO 8601 string like "2026-07-07T12:30:00Z". A date on its own, like "2026-07-07", is read as midnight UTC.
datetime.parse(text, format, utc?)From text in a custom format. See the format codes below.
datetime.fromComponents(c, utc?)From a table with year and optional month, day, hour, minute, second, millisecond.
datetime.isDateTime(value)True if the value is a DateTime object.

A DateTime is one exact moment, stored in UTC. Only reading it out as text or components involves a time zone. Wherever a function takes utc, leaving it off means your local time zone and passing true means UTC.

src/main.luau
--#include[datetime]

local launch = datetime.fromComponents({ year = 2026, month = 7, day = 7, hour = 9 })
local meeting = datetime.parse("07/07/2026 14:30", "%d/%m/%Y %H:%M")
print(datetime.now())  -- prints as ISO, like 2026-07-07T19:04:05.123Z

Reading it back out

MethodReturnsDescription
dt:toUnix()numberSeconds since the Unix epoch, with fractions.
dt:toMillis()numberMilliseconds since the Unix epoch.
dt:toISO()stringAn ISO 8601 string in UTC, like "2026-07-07T12:30:00.000Z". tostring(dt) gives the same.
dt:format(format?, utc?)stringText using the format codes below. The default format is "%Y-%m-%d %H:%M:%S".
dt:components(utc?)tableThe parts as numbers; see below.
dt:offset()numberYour local zone's distance from UTC at that moment, in seconds.

Components

dt:components(utc?) returns a table with year, month (1-12), day, hour, minute, second, millisecond, weekday (1 is Sunday, like os.date), yearday, and offset (seconds from UTC).

Format codes

The common strftime codes, the same ones os.date uses.

CodeMeaningExample

Math and comparing

DateTimes work with the normal operators. Subtracting two gives the seconds between them; adding or subtracting a number shifts by that many seconds; comparisons order them in time.

ExpressionResult
a - bnumber — seconds from b to a, negative if a is earlier
dt + 60, dt - 60a new DateTime, one minute later or earlier
a < b, a <= b, a == bboolean — order in time
dt:add(delta, utc?)a new DateTime, shifted by calendar units

dt:add takes a table with any of years, months, days, hours, minutes, seconds, and milliseconds, each possibly negative. Unlike adding raw seconds it understands the calendar: a month past January 31st lands on the last day of February, and days respect daylight saving changes in your local zone.

src/main.luau
local start = datetime.now()
doSomethingSlow()
print(("took %.2f seconds"):format(datetime.now() - start))

local due = datetime.now():add({ months = 1 })
if datetime.now() < due then
    print("due " .. due:format("%B %d"))
end
Errors

Bad input raises a Lua error: text that does not match the format in datetime.parse, an invalid format code, impossible components like month 13, or adding two DateTimes together. Wrap calls in pcall when the input comes from outside your program.
