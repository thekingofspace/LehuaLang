Library Reference

The net library

The net library is Lehua's networking toolbox. It covers HTTP requests and serving, raw TCP and UDP, local IPC channels, DNS lookups, WebSocket clients and hosts, and FTP with a sink type for streaming files. Everything is async and cooperative, so a program can talk to many things at once on a single thread.

Including net

--#include[net]

Events and receivers

Every socket, connection, and server in net can push its work to you through events. You bind a receiver with on(event, callback) and the runtime calls it every time something arrives: a new connection on a server, a chunk of data on a TCP connection, a datagram on a UDP socket, a request on an HTTP server. This is the recommended way to host things, because nothing sits in a loop calling a receive function over and over.

Three rules make receivers predictable:

RuleWhat it means
Every callback runs in its own coroutineEach event spawns a fresh coroutine for the callback, so a slow handler never blocks the next event. Ten clients can be inside your connection handler at once. Handlers can freely call async things like net.http.get or socket reads and writes.
One receiver per eventBinding the same event twice on one object raises an error. On a stream connection, "data" and "line" count as the same slot because both consume the incoming bytes.
Receivers keep the program aliveWhen your main script reaches its end, Lehua keeps running as long as any receiver is still bound. Call close() on the server or connection to release it and let the program exit.
src/main.luau
--#include[net]

local server = net.tcp.listen("127.0.0.1", 4000)
print("listening on " .. server:address())

server:on("connection", function(client)
    client:on("line", function(line)
        client:write("you said: " .. line .. "\n")
    end)
    client:on("close", function()
        print("client left")
    end)
end)

If a handler raises an error, the error is printed to stderr and the rest of the program keeps running. An HTTP handler that errors also answers its client with a 500 so nobody is left hanging.

Polling instead

The pull style still works everywhere: call accept() on a server or read(), readLine(), recv(), receive() on a socket to wait for the next thing yourself. It is fine for simple scripts and clients, but do not mix polling and a receiver on the same object; they would compete for the same incoming bytes.

Hosts and ports

Every function that binds or connects takes a host and a port, so here is what they mean once, for all of them.

ValueMeaning
host when listeningWhich of the machine's addresses to accept traffic on. "0.0.0.0" (the default) means every interface, reachable from other machines. "127.0.0.1" means only programs on the same machine can connect. You can also bind a specific interface's IP.
host when connectingWhere to connect to: an IP address like "93.184.216.34" or a name like "example.com", which is resolved through DNS for you.
portA number from 1 to 65535. Ports below 1024 usually need administrator rights to listen on.
port of 0When listening, asks the OS to pick a free port. Read the real one back with address() or localAddress(), which return "ip:port" strings.

HTTP requests

net.http.request makes a request and waits for the whole response. Pass a URL and an options table, or a single options table with a url field. The helpers net.http.get, post, put, patch, delete, head, and options set the method for you.

OptionMeaning
methodThe HTTP method. Defaults to "GET".
headersA table of request headers.
queryA table of values added to the URL as a query string.
bodyA string or buffer sent as the body.
jsonA value encoded as JSON. Sets the content type for you.
formA table sent as a URL encoded form.
timeoutSeconds to wait before giving up.
auth{ username = ..., password = ... } for basic auth, or { bearer = "token" }.
redirectsA number to cap redirects, or false to follow none.
insecuretrue to accept invalid TLS certificates. Use with care.

The response is a table with ok (true for a 2xx status), status, statusText, headers (keyed by lower case name), body, and the final url after any redirects.

src/main.luau
--#include[net, serde]

local res = net.http.get("https://api.example.com/status")
if res.ok then
    local data = serde.decode("json", res.body)
    print(data.uptime)
end

net.http.post("https://api.example.com/items", {
    json = { name = "widget", count = 3 },
    auth = { bearer = os.getenv("API_TOKEN") },
    timeout = 10,
})

HTTP serving

net.http.serve(port) or net.http.serve({ host = ..., port = ... }) starts an HTTP server and returns an HttpServer. The host defaults to "0.0.0.0". Connections are accepted and parsed in the background, so many clients can be in flight at once.

HttpServerDescription
server:on("request", fn)Binds the request receiver. The callback gets an HttpIncoming and runs in its own coroutine per request.
server:accept()Polling alternative: waits for the next request and returns it, or nil once the server is closed.
server:address()The bound "ip:port" string.
server:close()Stops accepting, ends the receiver, and releases the port.

The request callback can answer in two ways. Return a string or a response table and the server sends it for you. Or call request:respond(...) yourself somewhere in the handler and return nothing. If the handler returns nothing and never responded, the client gets a 500 instead of waiting forever; the same happens if the handler errors.

HttpIncomingReturns
request:method()The HTTP method, like "GET".
request:path()The path without the query string.
request:target()The full request target, path plus query string.
request:query()The raw query string, or "".
request:headers()A table of headers, keyed by lower case name.
request:header(name)One header value, or nil.
request:body()The request body as a string.
request:peerAddress()The client's address.
request:respond(res)Sends the reply. Call it once per request.

respond and the returned value take the same shapes: a plain string, which is sent as a 200 with a plain text type, or a table with status, headers, and a body string or json value.

src/main.luau
--#include[net]

local server = net.http.serve(8080)
print("http://" .. server:address())

server:on("request", function(req)
    if req:path() == "/health" then
        return { status = 200, json = { ok = true } }
    end
    return { status = 404, body = "not found" }
end)
Serving TLS

The server speaks plain HTTP. To put it behind HTTPS, run it behind a reverse proxy like nginx or Caddy that holds the certificate and forwards to your port.

TCP

net.tcp.connect(host, port) opens a connection to a server and returns a TcpConnection. net.tcp.listen(host?, port) binds a TcpServer; the host defaults to "0.0.0.0".

TcpServerDescription
server:on("connection", fn)Binds the connection receiver. The callback gets a TcpConnection for every client that connects, each in its own coroutine.
server:accept()Polling alternative: waits for the next client, or returns nil once the server is closed.
server:address()The bound "ip:port" string.
server:close()Stops listening. Wakes any pending accept with nil and ends the connection receiver. Connections already made stay open.

A TcpConnection is the same type on both sides: the one connect returns and the one a server hands to you. Its events and methods:

TcpConnectionDescription
conn:on("data", fn)Calls fn(chunk) with each chunk of bytes as it arrives.
conn:on("line", fn)Calls fn(line) once per line, with the newline stripped. Use either "data" or "line", not both.
conn:on("close", fn)Calls fn(err) once when the connection ends. err is nil for a clean end, or a message when it broke. Binding after the connection already ended fires immediately.
conn:read(count?)Reads up to count bytes, or one chunk if left out. nil at the end.
conn:readLine()Reads one line without the newline. nil at the end.
conn:readUntil(delimiter)Reads up to a one byte delimiter, which is dropped.
conn:readAll()Reads until the other side closes.
conn:write(data)Sends a string or buffer.
conn:peerAddress()The far end's "ip:port".
conn:localAddress()This end's "ip:port".
conn:close()Closes the connection, stops its receivers, and fires the "close" event.
src/main.luau
--#include[net]

local conn = net.tcp.connect("example.com", 7)
conn:on("data", function(chunk)
    print("echoed back: " .. chunk)
    conn:close()
end)
conn:write("hello")

UDP

net.udp.bind(host?, port?) makes a UdpSocket. The host defaults to "0.0.0.0" and the port to 0, a free one picked by the OS. UDP has no connections; you send to and receive from addresses directly, one datagram at a time.

UdpSocketDescription
sock:on("message", fn)Calls fn(data, ip, port) for every datagram that arrives, each in its own coroutine.
sock:sendTo(data, host, port)Sends one datagram, returns bytes sent.
sock:recvFrom()Polling alternative: waits for a datagram and returns the data, the sender's IP, and its port.
sock:connect(host, port)Fixes a default peer so you can use send and recv.
sock:send(data)Sends to the connected peer, returns bytes sent.
sock:recv()Receives from the connected peer.
sock:broadcast(on?)Allows sending to a broadcast address.
sock:localAddress()The socket's own "ip:port".
sock:close()Closes the socket and ends its message receiver. Further calls on it raise an error.
src/main.luau
--#include[net]

local sock = net.udp.bind("0.0.0.0", 9000)
sock:on("message", function(data, ip, port)
    sock:sendTo("echo: " .. data, ip, port)
end)

IPC

IPC is like TCP but for talking between programs on the same machine, with no network involved. It uses named pipes on Windows and Unix domain sockets elsewhere. You give a channel a name; both sides use the same name.

FunctionReturns
net.ipc.connect(name)An IpcConnection to a listener with that name.
net.ipc.listen(name)An IpcServer.

A plain name like "my-app" is a private, machine wide channel. A name with a slash, like "/tmp/my.sock", is treated as a socket file path.

IpcServerDescription
server:on("connection", fn)Binds the connection receiver. The callback gets an IpcConnection per client.
server:accept()Polling alternative: the next connection, or nil once closed.
server:name()The channel name it was created with.
server:close()Stops listening and ends the receiver.
IpcConnectionDescription
conn:on("data", fn)Calls fn(chunk) per chunk of bytes.
conn:on("line", fn)Calls fn(line) per line. One of "data" or "line", not both.
conn:on("close", fn)Calls fn(err) once when the connection ends.
conn:read(count?)Reads up to count bytes, or one chunk. nil at the end.
conn:readLine()Reads one line. nil at the end.
conn:write(data)Sends a string or buffer.
conn:close()Closes the connection, stops its receivers, and fires "close".
Close your IPC connections

Windows named pipes have no half close, so a receiver on an IPC connection cannot tell the other side is done until the connection actually closes. Call conn:close() when a conversation is over, or a bound receiver will keep the program alive waiting for more data.

src/main.luau
--#include[net]

-- server.luau
local server = net.ipc.listen("my-app")
server:on("connection", function(conn)
    conn:on("line", function(line)
        print("got: " .. line)
        conn:close()
    end)
end)

-- client.luau
local conn = net.ipc.connect("my-app")
conn:write("hello from the other process\n")

DNS

net.dns looks names up through the system's resolver.

FunctionReturnsDescription
net.dns.lookup(host){ string }Every IP address for a host.
net.dns.lookupOne(host)string?Just the first address, or nil.
net.dns.reverse(ip){ string }The host names that point at an IP.
net.dns.resolve(name, type){ any }Records of a type; see below.

resolve understands "A" and "AAAA" (arrays of IP strings), "MX" (tables with priority and host), "TXT" (arrays of strings), "SRV" (tables with priority, weight, port, target), "NS" (arrays of names), and other types like "CNAME" as arrays of text.

src/main.luau
--#include[net]

for _, ip in net.dns.lookup("example.com") do
    print(ip)
end

for _, mx in net.dns.resolve("example.com", "MX") do
    print(mx.priority, mx.host)
end

WebSocket

net.ws.connect(url) opens a client to a ws:// or wss:// server and returns a WebSocket. net.ws.listen(host?, port) starts a WebSocketServer; the host defaults to "0.0.0.0".

WebSocketServerDescription
server:on("connection", fn)Binds the connection receiver. The callback gets a WebSocket for every client that finishes the handshake.
server:accept()Polling alternative: the next upgraded connection, or nil once closed.
server:address()The bound "ip:port" string.
server:close()Stops listening and ends the receiver.

A WebSocket is the same type on both sides:

WebSocketDescription
ws:on("message", fn)Calls fn(msg) with a message table for every text, binary, ping, or pong message.
ws:on("close", fn)Calls fn(code, reason) once when the socket ends. code and reason come from the close frame when there is one; on a transport error code is nil and reason carries the error text.
ws:send(text)Sends a text message.
ws:sendBinary(data)Sends a binary message from a string or buffer.
ws:ping(data?)Sends a ping.
ws:receive()Polling alternative: waits for the next message table. nil when the socket ends.
ws:close(code?, reason?)Sends a close frame, stops the receivers, and fires "close".
ws:peerAddress()The far end's address, or the URL it connected to.

A message is a table with a type of "text", "binary", "ping", or "pong". Text and binary messages carry their bytes in data. When polling with receive, a close frame also shows up as a "close" message with an optional code and reason.

src/main.luau
--#include[net]

local server = net.ws.listen("127.0.0.1", 8081)
server:on("connection", function(ws)
    ws:on("message", function(msg)
        if msg.type == "text" then
            ws:send("echo: " .. msg.data)
        end
    end)
end)

local client = net.ws.connect("ws://127.0.0.1:8081/")
client:on("message", function(msg)
    print("got: " .. msg.data)
    client:close(1000, "done")
    server:close()
end)
client:send("hello")

FTP and sinks

net.ftp.connect opens an FTP session. Pass a table with host and the optional port (21 by default), user, and password.

FtpSessionReturnsDescription
ftp:pwd()stringThe current directory.
ftp:cwd(path)nilChanges directory. ftp:cdup() goes up one.
ftp:list(path?){ string }A detailed listing. ftp:nameList(path?) gives just names.
ftp:mkdir(path)nilMakes a directory. removeDir removes one.
ftp:remove(path)nilDeletes a file.
ftp:rename(from, to)nilRenames or moves.
ftp:size(path)numberA file's size in bytes.
ftp:download(remote, into?)string | numberDownloads a file. See sinks below.
ftp:downloadTo(remote, localPath)stringDownloads straight to a local file.
ftp:upload(remote, source)numberUploads a string, buffer, or sink.
ftp:uploadFile(remote, localPath)numberUploads a local file.
ftp:close()nilEnds the session politely.

Sinks

A sink is a place to pour bytes into as they arrive, then turn into a real file when you are ready. net.sink() makes an in memory sink; net.sink({ file = path }) makes one backed by a file on disk, which is the right choice for large downloads that should not sit in memory. The path follows the same rules as the fs library, so @self and friends work.

SinkReturnsDescription
sink:write(data)nilAppends a string or buffer.
sink:bytes()stringEverything gathered so far.
sink:buffer()bufferThe contents as a Luau buffer.
sink:len()numberHow many bytes are in the sink. #sink works too.
sink:save(path)stringWrites the contents to a real file and returns its path.
sink:clear()nilEmpties the sink.
sink:isFile()booleanTrue for a file backed sink.
sink:path()string?The backing file path, or nil for a memory sink.

Passing a sink to ftp:download pours the remote file into it and returns the byte count instead of the data. Then sink:save writes it to a real file. This keeps a big transfer off the Lua side until you decide where it goes.

src/main.luau
--#include[net]

local ftp = net.ftp.connect({ host = "ftp.example.com", user = "me", password = "secret" })

-- Small file: get the bytes directly.
local readme = ftp:download("README.txt")

-- Large file: stream it into a file backed sink, then save it.
local sink = net.sink({ file = "@self/downloads/big.zip" })
ftp:download("archive/big.zip", sink)
local saved = sink:save("@self/downloads/big.zip")
print("saved " .. #sink .. " bytes to " .. saved)

ftp:upload("upload/notes.txt", "written from lehua\n")
ftp:close()
Errors

Every net call raises a Lua error when something goes wrong, such as a refused connection, a DNS miss, or an FTP command that fails. Wrap calls in pcall when you want to handle trouble yourself. Errors raised inside an event callback are printed to stderr and do not stop the program; on a stream connection a broken transport reaches you through the "close" event's err argument.
