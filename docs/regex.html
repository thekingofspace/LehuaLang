Library Reference

The regex library

The regex library brings real regular expressions to Lehua. You compile a pattern once with regex.new, then use the same object to test, search, capture, replace, and split.

Including regex

--#include[regex]
This is not Luau pattern syntax

Luau's built in string.match and friends use Luau patterns, where a digit is %d. The regex library uses regular expression syntax, where the same thing is \d. The two look alike but are different languages, and a pattern written for one usually misbehaves in the other.

Writing patterns

Patterns use Rust's regex syntax. It has the pieces most regex flavors share: character classes like \d and [a-z], quantifiers like + and {2,5}, the anchors ^ and $, alternation with |, and capture groups in parentheses. It has no backreferences or lookaround; in exchange, matching stays fast no matter what input it is handed.

Backslashes are special inside quoted Luau strings, so long strings are the comfortable way to write a pattern: [[\d+]] instead of "\\d+".

Compiling a pattern

regex.new(pattern, flags?) compiles a pattern and returns a Regex object. The second argument is a string of one letter flags; leave it off for none. A pattern that does not compile, or a letter outside this table, raises a Lua error.

FlagNameEffect
"i"ignore caseLetters match both upper and lower case.
"m"multi line^ and $ also match at the start and end of each line, not just of the whole text.
"s"dot all. also matches line breaks.
"x"verboseWhitespace in the pattern is ignored and # starts a comment, so a long pattern can be laid out over several lines.
"U"swap greedQuantifiers like * match as little as possible, and the lazy forms like *? as much as possible.
src/main.luau
--#include[regex]

local version = regex.new([[(\d+)\.(\d+)\.(\d+)]])
print(version:isMatch("lehua 0.1.1"))  -- true

local heading = regex.new([[^#\s+.+$]], "im")
print(tostring(heading))  -- Regex(^#\s+.+$)

One-off helpers

FunctionDescription
regex.escape(text)Returns the text with every regex special character escaped, so it can sit inside a pattern and match itself literally. regex.escape("1+1") is "1\+1".
regex.isMatch(pattern, text)Compiles the pattern with no flags and reports whether it matches anywhere in the text. Handy for a single check, but it compiles the pattern every call — inside a loop, compile once with regex.new instead.

Match tables

Wherever a method reports a match it uses a small table with three fields.

FieldMeaning
textThe matched text.
startThe position of its first byte, counting from 1.
finishThe position of its last byte, inclusive.

The positions work exactly like string.find: text:sub(m.start, m.finish) gives back m.text.

Searching

MethodReturnsDescription
re:isMatch(text)booleanTrue if the pattern matches anywhere in the text. The cheapest question to ask.
re:find(text, init?)match table or nilThe first match, or nil if there is none. init is the position to start looking from; the default is 1.
re:findAll(text){ match table }Every non-overlapping match, left to right. Empty when nothing matches.

With init, anchors keep their meaning: ^ still means the start of the whole string even when the search begins past it. An init beyond the end of the text makes find return nil.

src/main.luau
--#include[regex]

local number = regex.new([[\d+]])
local text = "call me at 555-0134"

local m = number:find(text)
print(m.text, m.start, m.finish)  -- 555   12   14

for _, hit in number:findAll(text) do
    print(hit.text)  -- 555, then 0134
end

Capture groups

Parentheses in a pattern capture the part of the text they matched. re:captures(text) returns the first match with all of its groups, or nil when there is no match. re:capturesAll(text) returns an array of the same tables, one per match.

KeyValue
.matchA match table for the whole match.
[1], [2], …A match table for each group, in pattern order — or false when that group was not part of the match, for example the unused side of a |.
["name"]Groups named with (?<name>...) also appear under their name. A named group that did not match is simply absent.
src/main.luau
--#include[regex]

local kv = regex.new([[(?<key>\w+)\s*=\s*(?<value>\S+)]])

local caps = kv:captures("retries = 3")
if caps then
    print(caps.match.text)                 -- retries = 3
    print(caps.key.text, caps.value.text)  -- retries   3
    print(caps[1].text)                    -- retries (named groups count by position too)
end

local answer = regex.new([[(yes)|(no)]])
print(answer:captures("no")[1])  -- false: the first group sat this match out

Replacing

MethodDescription
re:replace(text, replacement, limit?)Returns the text with at most limit matches replaced. The limit defaults to 1, so a plain call changes only the first match; a limit of 0 changes nothing. Use replaceAll to replace every match.
re:replaceAll(text, replacement)Returns the text with every match replaced.

The replacement string can refer back to the match.

WrittenInserts
$0The whole match.
$1, $2, …That numbered group.
$name or ${name}That named group.
$$A literal dollar sign.

A group that did not take part inserts nothing. Use the braced form when the reference runs into other letters: in "$1px" the library looks for a group named 1px, while "${1}px" means group 1 followed by px.

src/main.luau
--#include[regex]

local date = regex.new([[(\d{4})-(\d{2})-(\d{2})]])
print(date:replaceAll("from 2026-07-07 to 2026-08-01", "$3/$2/$1"))
-- from 07/07/2026 to 01/08/2026

local pair = regex.new([[(?<a>\w+) (?<b>\w+)]])
print(pair:replace("one two three four", "${b} ${a}"))
-- two one three four  (replace only touches the first match)

Splitting

re:split(text, limit?) cuts the text at every match and returns the pieces between them as an array; the matches themselves are dropped. With a limit of n the result has at most n pieces, and the last one keeps the rest of the text unsplit. A match at the very start of the text, or two matches back to back, produce empty strings — that is how you can tell a field was empty.

src/main.luau
--#include[regex]

local comma = regex.new([[\s*,\s*]])
for _, part in comma:split("one, two ,three") do
    print(part)  -- one, then two, then three
end

local parts = comma:split("a, b, c", 2)
print(parts[1], parts[2])  -- a   b, c

Bytes and positions

Luau strings are strings of bytes, and the regex library matches on those bytes directly. start and finish count bytes, so a character that takes several bytes in UTF-8, like é, advances the count by more than one. Classes such as \w and . still understand UTF-8 text, so they match whole characters rather than splitting one down the middle.

Errors

regex.new and regex.isMatch raise a Lua error for a pattern that does not compile, or for a flag outside i m s x U. Wrap them in pcall when the pattern comes from outside your program. Methods on a compiled Regex do not raise — a search that finds nothing returns nil or an empty array.
