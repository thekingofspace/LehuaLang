Library Reference

The url library

The url library takes web addresses apart and puts them together. It parses a URL into its pieces, resolves links the way a browser does, and handles the percent encoding that makes text safe to put inside an address.

Including url

--#include[url]

Parsing a URL

url.parse(text) splits a URL into a table of named parts. The text must be an absolute URL, meaning it starts with a scheme like https:. Anything else, such as "/api/users" or "example.com", raises a Lua error, so wrap the call in pcall when the text comes from outside your program.

FieldTypeWhen it is present
hrefstringAlways. The whole URL, cleaned up: the scheme and host are lowercased and a missing path becomes /.
schemestringAlways. "https", "ftp", and so on, without the colon.
hoststring?Only when the URL has one. A mailto: link, for example, has no host.
portnumber?Only when the URL spells out a port that is not the default for its scheme. https://example.com:443/ has no port field because 443 is what https uses anyway.
pathstringAlways. "/" when the URL has no path of its own.
usernamestring?Only when the URL includes one, as in ftp://user@host/.
passwordstring?Only when the URL includes one.
querystring?Only when the URL has a ? part. The raw text after the ?, still percent encoded.
queryParamstable?Present whenever query is. The query split into a table of decoded key and value strings. If a key appears more than once, the last value wins.
fragmentstring?Only when the URL has a # part. The text after the #.
src/main.luau
--#include[url]

local parts = url.parse("https://user:secret@example.com:8080/search?q=lehua+lang&page=2#results")

print(parts.scheme)              -- https
print(parts.host)                -- example.com
print(parts.port)                -- 8080
print(parts.path)                -- /search
print(parts.username)            -- user
print(parts.password)            -- secret
print(parts.query)               -- q=lehua+lang&page=2
print(parts.queryParams.q)       -- lehua lang
print(parts.queryParams.page)    -- 2
print(parts.fragment)            -- results
Note

Optional fields are simply missing from the table, not set to some empty value. Check them with if parts.port then. Values in queryParams are always strings, even when they look like numbers.

Joining URLs

url.join(base, reference) resolves a link against a page, the same way a browser resolves an href. The base must be an absolute URL. The reference can be relative, like "guide.html" or "../logo.png", or absolute, in which case it simply replaces the base. The result comes back as a string. A base that cannot be parsed raises a Lua error.

src/main.luau
--#include[url]

print(url.join("https://example.com/docs/intro.html", "guide.html"))
-- https://example.com/docs/guide.html

print(url.join("https://example.com/docs/", "../images/logo.png"))
-- https://example.com/images/logo.png

print(url.join("https://example.com/docs/", "https://other.site/page"))
-- https://other.site/page

Checking validity

url.isValid(text) returns true when url.parse would succeed and false when it would raise. Use it to test user input without a pcall. Remember that only absolute URLs pass, so "example.com" without a scheme returns false.

src/main.luau
print(url.isValid("https://example.com/"))  -- true
print(url.isValid("example.com/page"))      -- false, no scheme
print(url.isValid("not a url"))             -- false

Encoding one piece of text

Percent encoding replaces characters that have special meaning in a URL with % followed by two hex digits, so a space becomes %20 and & becomes %26. Do this whenever you drop free text into a URL you are building by hand, otherwise a stray & or ? changes what the URL means.

FunctionWhat it does
url.encode(text)Percent encodes one component, like JavaScript's encodeURIComponent. Letters, digits, and - _ . ~ pass through; everything else is encoded.
url.decode(text)Reverses it, turning every %XX sequence back into the character it stands for. It does not touch +; that is a query string habit handled by url.decodeQuery.
src/main.luau
--#include[url]

local term = "cats & dogs?"
print("https://example.com/search?q=" .. url.encode(term))
-- https://example.com/search?q=cats%20%26%20dogs%3F

print(url.decode("cats%20%26%20dogs%3F"))
-- cats & dogs?
Note

url.encode is for one piece at a time, such as a single query value or one path segment. Do not run a whole URL through it; the / and ? that hold the URL together would get encoded too.

Query strings

A query string is the key=value&key=value part after the ?. Two functions convert between that text and a Luau table.

url.encodeQuery(params) takes a table with string keys and returns the encoded query, without a leading ?. Values may be strings, numbers, or booleans; numbers and booleans are written out as text, and any other type raises a Lua error that names the offending key. Keys come out sorted alphabetically, so the same table always produces the same string. Spaces are written as +, which is the traditional form for query strings.

url.decodeQuery(text) goes the other way and returns a table of string keys and string values. A leading ? is ignored, + becomes a space, and percent encoding is undone. If a key appears more than once, the last value wins. It never raises.

src/main.luau
--#include[url]

local query = url.encodeQuery({ q = "lehua lang", page = 2, exact = true })
print(query)
-- exact=true&page=2&q=lehua+lang

print("https://example.com/search?" .. query)

local params = url.decodeQuery("?q=lehua+lang&page=2")
print(params.q)     -- lehua lang
print(params.page)  -- 2, as the string "2"
