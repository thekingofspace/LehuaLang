Library Reference

The canvas library

The canvas library creates and edits images in memory. A canvas is an RGBA bitmap you draw shapes, gradients, text, and other images onto, run filters over, and encode to PNG, JPG, and friends. Together with fs.readImage and fs.writeImage it covers the whole pipeline from a blank bitmap to a finished file.

Including canvas

--#include[canvas]

Creating canvases

FunctionReturnsDescription
canvas.new(width, height, background?)CanvasA new canvas up to 16384x16384. The background defaults to transparent.
canvas.decode(bytes)CanvasDecodes encoded image bytes (from a file, an HTTP body, anywhere).
canvas.fromPixels(width, height, rgba)CanvasBuilds a canvas from raw RGBA bytes, 4 per pixel.
canvas.font(path)FontLoads a TTF or OTF font for text, e.g. "C:/Windows/Fonts/arial.ttf". font:measure(text, size) returns the pixel width and height.
canvas.formats(){ string }The encodable formats: png, jpg, gif, bmp, ico, tiff, webp, qoi, tga.

Colors

Every color argument accepts three shapes, and channel values run 0 to 255:

FormExample
Hex string, 3, 4, 6, or 8 digits"#f80", "#ff8800", "#ff880080" (last two digits are alpha)
Named color"red", "navy", "orange", "transparent", and the other common CSS names
Table{ r = 255, g = 136, b = 0, a = 128 } or positional { 255, 136, 0 }

The helpers canvas.rgb(r, g, b), canvas.rgba(r, g, b, a), canvas.hsv(h, s, v, a?) (hue 0-360, s and v 0-1), and canvas.hex(str) build color tables for you. A color with alpha below 255 blends with what is already on the canvas, so translucent panels and overlays work the way you expect.

Reading and writing pixels

MethodReturnsDescription
c:width() / c:height() / c:size()numberDimensions in pixels.
c:clone()CanvasAn independent copy.
c:getPixel(x, y){r, g, b, a}One pixel. Coordinates start at 0 in the top left.
c:setPixel(x, y, color)CanvasWrites one pixel. Out of bounds writes are ignored.
c:pixels()stringThe raw RGBA bytes.
c:mapPixels(fn)CanvasCalls fn(x, y, r, g, b, a) for every pixel; returned values replace the channels.

c:floodFill(x, y, color, tolerance?)CanvasPaint bucket from a point, spreading over pixels within tolerance of the start color.

Every drawing and editing method returns the canvas itself, so calls chain: canvas.new(64, 64):fill("navy"):circle(32, 32, 20, "gold"):blur(2).

Shapes

Coordinates are pixels; angles are degrees. Where thickness appears it defaults to 1 and thick strokes get round caps.

MethodDescription
c:line(x1, y1, x2, y2, color, thickness?)A straight line.
c:dashedLine(x1, y1, x2, y2, color, thickness?, dash?, gap?)A dashed line. The dash length defaults to 6, and the gap defaults to the dash length.
c:polyline(points, color, thickness?)Connected line segments.
c:rect(x, y, w, h, color)A filled rectangle. strokeRect draws just the border.
c:roundedRect(x, y, w, h, radius, color)A filled rectangle with rounded corners. strokeRoundedRect(..., thickness?) for the border.
c:circle(cx, cy, r, color)A filled circle. strokeCircle(..., thickness?) for the ring.
c:ellipse(cx, cy, rx, ry, color)A filled ellipse. strokeEllipse for the outline.
c:polygon(points, color)A filled polygon. strokePolygon(..., thickness?) for the outline.
c:triangle(x1, y1, x2, y2, x3, y3, color)A filled triangle.
c:arc(cx, cy, r, startAngle, endAngle, color, thickness?)A circular arc stroke.
c:pie(cx, cy, r, startAngle, endAngle, color)A filled wedge.
c:bezier(x1, y1, cx1, cy1, cx2, cy2, x2, y2, color)A cubic bezier curve.

points is either a flat list { x1, y1, x2, y2, ... } or a list of { x, y } tables.

Gradients

Gradients repaint the whole canvas. To fill a region, paint a small canvas and drawImage it where you want it. Stops are a list of colors spread evenly, or entries with explicit positions: { {0, "red"}, {0.8, "blue"}, {offset = 1, color = "white"} }.

MethodDescription
c:linearGradient(x1, y1, x2, y2, stops)Blends along the line from one point to the other.
c:radialGradient(cx, cy, radius, stops)Blends outward from a center.
c:conicGradient(cx, cy, startAngle?, stops)Blends around a center, like a color wheel.

Text

c:text(text, x, y, size, color, font, options?) draws text with a font from canvas.font. Newlines start new lines. Options: spacing adds extra pixels between letters (the classic spaced heading look), lineHeight overrides the default of 1.2 times the size.

src/main.luau
--#include[canvas, fs]

local font = canvas.font("C:/Windows/Fonts/arial.ttf")
local card = canvas.new(400, 120, "#0a0e1e")
card:text("VIRETH CULTIVATION", 20, 30, 28, "#9be352", font, { spacing = 4 })
card:dashedLine(20, 80, 380, 80, "#334466", 1, 4, 4)
fs.writeImage("@self/title.png", card)

Layers and merging

Any canvas can be drawn onto any other, which is how you build images in layers: compose each piece on its own transparent canvas, then stack them.

MethodDescription
c:drawImage(other, x, y, options?)Draws another canvas at a position with alpha blending. Options: opacity 0-1, blend mode, width/height to scale it on the way in, filter for the scaling quality.
c:merge(other, blend?, opacity?)Blends another canvas over this one at the origin, like a layer merge in an image editor.
c:tile(other)Repeats another canvas across this one as a pattern.
c:mask(other, mode?)Uses another canvas as an alpha mask: bright (or opaque, with mode "alpha") areas keep pixels, dark areas erase them.

The blend modes are "normal", "add", "subtract", "multiply", "screen", "overlay", "darken", "lighten", "difference", and "exclusion".

Shadows, glows, and outlines

These read the canvas's own alpha channel and paint underneath the existing content, so the workflow is: draw your shape or text on a transparent canvas, add the effect, then drawImage the whole layer onto the background.

MethodDescription
c:dropShadow(dx, dy, sigma, color?, strength?)A blurred shadow offset by dx, dy. Black by default.
c:glow(sigma, color?, strength?)A soft halo around everything opaque. White by default; pass the shape's own color for a neon look.
c:outline(color, radius?)A hard edge traced around everything opaque.
src/main.luau
--#include[canvas, fs]

local bg = canvas.new(200, 200, "#0a0e1e")

local badge = canvas.new(160, 160)
badge:strokeCircle(80, 80, 55, "#9be352", 3)
badge:circle(80, 80, 16, "#9be352")
badge:glow(8, "#9be352", 1.5)

bg:drawImage(badge, 20, 20)
fs.writeImage("@self/badge.png", bg)

Noise maps

canvas.noise(options?) creates a seeded, endless field of coherent noise for textures, terrain, clouds, and generative art. The field is defined for every coordinate, so you can render one region now and a neighboring region later and they join seamlessly, which makes maps expandable after creation. All options can be changed later with set, letting you interject steps mid-generation for unique results.

OptionDefaultMeaning
seedrandomThe same seed always produces the same field.
scale64Pixels per feature; bigger means smoother, broader shapes.
octaves4Layers of detail, 1 to 12.
persistence0.5How much each octave contributes; higher is rougher.
lacunarity2Frequency step between octaves.
kind"perlin""perlin" (smooth), "value" (blocky-smooth), "worley" (cellular), "ridged" (mountain ridges).
MethodDescription
n:sample(x, y)The noise value at a point, 0 to 1. Same inputs always give the same value.
n:image(w, h, options?)Renders a region into a new canvas. Options: x/y world offset for rendering a different window of the field, stops to map values through a gradient (instant terrain coloring), alpha = true to write into the alpha channel instead.
n:fill(target, options?)Same, into an existing canvas.
n:set(options)Changes any option live, including the seed and kind.
n:warp(other, strength)Domain warping: another noise map bends this one's coordinates, producing organic, one-of-a-kind flows. n:unwarp() removes it.
src/main.luau
--#include[canvas, fs]

local terrain = canvas.noise({ seed = 1337, scale = 60, octaves = 6 })
local map = terrain:image(512, 512, {
    stops = {
        { 0.0, "#0b2d5c" },
        { 0.45, "#1d6fc2" },
        { 0.5, "#e8dfa8" },
        { 0.6, "#4d9440" },
        { 0.85, "#6b6b60" },
        { 1.0, "#ffffff" },
    },
})

-- Render the region just to the east later; the edge lines up seamlessly.
local east = terrain:image(512, 512, { x = 512 })

fs.writeImage("@self/map.png", map)

Saving and loading

FunctionReturnsDescription
c:encode(format?, quality?)stringThe encoded bytes, PNG by default. quality 1-100 applies to JPG.
c:dataUrl(format?)stringA data: URL for embedding in HTML.
fs.writeImage(path, canvas, format?, quality?)stringEncodes and writes a file. The format comes from the extension when not given. Returns the full path.

src/main.luau
--#include[canvas, fs]

local photo = fs.readImage("@self/photo.jpg")
photo:resize(800, 600)
photo:saturate(1.2)
photo:vignette(0.4)
fs.writeImage("@self/photo-edited.jpg", photo, "jpg", 92)
Performance

A canvas lives entirely in memory at 4 bytes per pixel, so a 4000x4000 image costs about 64 MB. Filters walk every pixel; on very large images prefer doing your edits before resize only when they need the detail, and use thumbnail for quick previews.
