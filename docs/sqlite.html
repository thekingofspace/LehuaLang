Library Reference

The sqlite library

The sqlite library gives every Lehua program a real SQL database in a single file, with no server to install. SQLite is built into the runtime, so a database is just a file next to your script.

Including sqlite

--#include[sqlite]

Opening a database

FunctionDescription
sqlite.fromLocal(path, options?)Opens a database file, creating it (and any missing folders) if needed. The path follows the same rules as fs: ./name, ../name, @self/name, @Alias/name, or an absolute path.
sqlite.fromConnection(info, options?)Opens from a raw connection string (":memory:" or a file: URI like "file:data.db?mode=ro"), or from a connection info table - see below.
sqlite.memory(options?)A fresh, empty database that lives in memory and disappears when it is closed.

Connection options

Every opener accepts an options table, and fromConnection can take one table that describes the whole connection. All fields are optional.

FieldMeaning
filefromConnection only: the database file, resolved like other paths. Leaving it out means an in-memory database.
readOnlyOpen without write access. Writes raise an error.
createWhether a missing file is created. Defaults to true; with false, opening a missing file raises an error.
cachefromConnection only: "shared" or "private" page cache.
immutablefromConnection only: promise SQLite the file cannot change (for media that is truly read only).
passwordThe encryption key of an SQLCipher database. The standard Lehua runtime ships plain SQLite, which has no password support, so this raises a clear error unless the runtime was built with SQLCipher.
busyTimeoutMsHow long to wait when another connection holds the lock before giving up.
foreignKeysTurn foreign key enforcement on or off.
journalMode"delete", "truncate", "persist", "memory", "wal", or "off". "wal" is the usual choice for apps with several readers.
cacheSizeKbThe page cache size in kilobytes.
pragmasA table of any other PRAGMA settings to apply, like { synchronous = "normal", user_version = 3 }.
src/main.luau
local db = sqlite.fromConnection({
    file = "@self/app.db",
    journalMode = "wal",
    foreignKeys = true,
    busyTimeoutMs = 5000,
    pragmas = { synchronous = "normal" },
})

local snapshot = sqlite.fromLocal("@self/app.db", { readOnly = true })

Inspecting a connection

db:info() returns everything there is to know about an open connection: path, readOnly, autocommit (false while inside a transaction), totalChanges, lastInsertId, sqliteVersion, journalMode, encoding, foreignKeys, busyTimeoutMs, pageSize, pageCount, cacheSize, userVersion, and schemaVersion.

src/main.luau
print(db:info())
-- prints the whole table, for example journalMode = "wal", pageSize = 4096, ...
src/main.luau
--#include[sqlite]

local db = sqlite.fromLocal("@self/app.db")
db:executeBatch([[
    CREATE TABLE IF NOT EXISTS notes (
        id INTEGER PRIMARY KEY,
        body TEXT NOT NULL,
        created DATETIME
    );
]])

Running SQL

MethodReturnsDescription
db:execute(sql, params?)numberRuns one statement that changes data. Returns how many rows were affected.
db:executeBatch(sql)nilRuns several statements separated by semicolons. No parameters.
db:query(sql, params?){ row }Runs a SELECT and returns every row as a table keyed by column name.
db:queryOne(sql, params?)row?Like query but returns just the first row, or nil if there is none.
db:lastInsertId()numberThe row id of the most recent insert.
db:changes()numberHow many rows the last statement changed.
db:tables(){ string }The names of the tables in the database.
db:info()tableEverything about the connection; see above.
db:isOpen()booleanFalse after close has been called.
db:close()nilCloses the database. Using it afterwards raises an error.

Parameters

Never build SQL by gluing strings together - pass values separately and let the database handle them. Use ? placeholders with an array, or named placeholders like :name with a table. Named placeholders may be written :name, @name, or $name in the SQL.

src/main.luau
db:execute("INSERT INTO notes (body, created) VALUES (?, ?)", { "hello", datetime.now() })

local rows = db:query("SELECT * FROM notes WHERE body LIKE :pat", { pat = "%hello%" })
for _, row in rows do
    print(row.id, row.body, row.created:format())
end

How values convert

Luau valueStored asComes back as
stringTEXT, or BLOB if it is not valid textstring
numberINTEGER or REALnumber
booleanINTEGER 0 or 1number (SQLite has no boolean type)
nilNULLthe key is simply absent from the row
a bufferBLOBstring
a DateTime objectTEXT in ISO 8601 forma DateTime object again

The DateTime round trip works through the column type: declare a column as DATETIME, DATE, or TIMESTAMP and anything stored there - ISO text, or a number of seconds since the Unix epoch - comes back as a DateTime object. Other columns are left alone.

Transactions

db:transaction(callback) runs the callback with all its changes grouped together. If the callback finishes, everything is committed at once. If it raises an error, everything is rolled back and the error is re-raised.

src/main.luau
db:transaction(function()
    db:execute("UPDATE accounts SET balance = balance - 50 WHERE id = ?", { 1 })
    db:execute("UPDATE accounts SET balance = balance + 50 WHERE id = ?", { 2 })
end)
Errors

Bad SQL, missing parameters, and constraint violations raise Lua errors with SQLite's message. Wrap calls in pcall to handle them yourself.
