Library Reference

The semver library

The semver library reads and compares semantic version numbers, the major.minor.patch scheme most software uses. Sorting version strings alphabetically gets 1.10.0 wrong; this library gets it right, and can answer questions like "does 1.4.2 satisfy ^1.2?".

Including semver

--#include[semver]

Version strings

A version must have all three numbers: major.minor.patch, like "1.4.2". Short forms such as "1.4" are not valid versions. Two optional parts can follow: a prerelease after a hyphen, like "2.0.0-beta.1", and build metadata after a plus sign, like "1.4.2+build.7". Leading and trailing spaces are ignored.

Parsing

semver.parse(version) splits a version string into a table. Invalid input raises a Lua error, so check with semver.isValid(version) first when the string comes from outside your program. isValid returns a boolean and never raises.

FieldTypeDescription
majornumberThe first number. Bump it for breaking changes.
minornumberThe second number. Bump it for new features.
patchnumberThe third number. Bump it for fixes.
prereleasestring?The part after the hyphen, like "beta.1". Only present when the version has one.
buildstring?The part after the plus sign. Only present when the version has one.
src/main.luau
--#include[semver]

local v = semver.parse("2.1.0-beta.1+build.7")
print(v.major, v.minor, v.patch)  -- 2 1 0
print(v.prerelease, v.build)      -- beta.1 build.7

print(semver.isValid("1.4.2"))  -- true
print(semver.isValid("1.4"))    -- false, needs all three numbers

Comparing

semver.compare(a, b) returns -1 when a is older, 0 when they are equal, and 1 when a is newer. The numbers compare as numbers, so 1.10.0 is newer than 1.9.0. A prerelease comes before its release: 1.0.0-rc.1 is older than 1.0.0. Build metadata is only used as a last tie breaker between otherwise equal versions.

src/main.luau
--#include[semver]

print(semver.compare("1.9.0", "1.10.0"))      -- -1
print(semver.compare("1.4.2", "1.4.2"))       -- 0
print(semver.compare("1.0.0", "1.0.0-rc.1"))  -- 1, the release is newer

Checking requirements

semver.satisfies(version, requirement) returns true when the version falls inside the range the requirement describes. Requirements use the same syntax as Cargo, Rust's package manager.

RequirementMatchesMeaning
^1.2>=1.2.0, <2.0.0Compatible releases: anything that only adds features or fixes. A bare version like 1.2.3 means the same as ^1.2.3.
~1.2.3>=1.2.3, <1.3.0Patch releases only.
>=1.2.3, <2the range itselfA list of comparators, separated by commas. Every one must hold. The comparators are =, >, >=, <, and <=.
=1.2.31.2.3 onlyExactly that version.
1.2.*>=1.2.0, <1.3.0Wildcard. * on its own matches any version.

Unlike versions, requirements may leave numbers off: ^1.2 and >=1.2, <2 are fine.

src/main.luau
--#include[semver]

print(semver.satisfies("1.4.2", "^1.2"))         -- true
print(semver.satisfies("2.0.0", "^1.2"))         -- false
print(semver.satisfies("1.2.9", "~1.2.3"))       -- true
print(semver.satisfies("1.5.0", ">=1.2.3, <2"))  -- true
Prereleases

A requirement without a prerelease never matches a prerelease version, so semver.satisfies("2.0.0-beta.1", "^2.0.0") is false. To opt in, put a prerelease on the same version in the requirement, like ">=2.0.0-0". Also, when major is 0 the caret is stricter: ^0.2.3 means >=0.2.3, <0.3.0, because 0.x releases make no compatibility promise.

Sorting

semver.sort(versions) takes an array of version strings and returns a new array ordered oldest to newest. The input is not changed. The returned strings are cleaned up, so any surrounding spaces are gone.

src/main.luau
--#include[semver]

local sorted = semver.sort({ "1.10.0", "1.2.0", "2.0.0-rc.1", "1.9.3" })
for _, v in ipairs(sorted) do
    print(v)
end
-- 1.2.0
-- 1.9.3
-- 1.10.0
-- 2.0.0-rc.1

local newest = sorted[#sorted]
Errors

An invalid version raises a Lua error in parse, compare, satisfies, and sort, and an invalid requirement raises one in satisfies. The message names the bad string. Use semver.isValid or pcall when the input comes from outside your program.
