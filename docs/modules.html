Modules and require

Every module is given its own require, parallel, __dirname, and __filename. You do not need to declare them. require understands a few forms:

  • require("./util") loads a file next to the current one. The .luau ending is added for you.
  • require("../shared/log") walks up to a parent folder.
  • require("./thing") where thing is a folder with an init.luau loads that init file.
  • require("@self/helper") points at the current module's own folder. This is handy inside an init.luau.
  • require("@root/hello") uses an alias from .luaurc.
  • require("hello") loads a Root package by name.

A module returns a value, and that value is cached. The next require of the same module gives back the same value.

src/main.luau
local util = require("./util")
local log = require("../shared/log")
local hello = require("hello")

print(util.add(2, 3))
print("this file lives in " .. __dirname)

Aliases come from .luaurc in the project root. The keys are alias names and the values are paths relative to that file.

.luaurc
{
    "languageMode": "nonstrict",
    "aliases": {
        "root": "./roots"
    }
}
Note

Luau requires the path in require to be a plain string. Lehua reads those strings when it builds, so it knows which files to pack. Keep require paths as plain strings and everything gets bundled.
