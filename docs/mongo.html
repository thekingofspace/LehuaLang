Library Reference

The mongo library

The mongo library talks to MongoDB. You connect to a cluster with a connection string, walk down to a database and a collection, and work with documents as plain Luau tables.

Including mongo

--#include[mongo]

Connecting

mongo.connect(connectionString, options?) returns a cluster object. Unlike sqlite, mongo always needs a connection string — there is no local file mode. The options table accepts serverSelectionTimeoutMs, connectTimeoutMs, and appName.

src/main.luau
--#include[mongo]

local cluster = mongo.connect("mongodb+srv://user:pass@cluster0.example.mongodb.net/myapp")
local db = cluster:defaultDatabase() or cluster:database("myapp")
local users = db:collection("users")
Connecting is lazy

connect only checks that the connection string makes sense. The first real operation reaches the server, and that is where a bad address or password shows up as an error. Set serverSelectionTimeoutMs if the default half minute wait is too long.

Find options

The second argument to find and findOne is a table:

OptionMeaning
sortWhich order to return documents in: { age = -1 } for one field, or an array of pairs like { {"age", -1}, {"name", 1} } when the order of the sort fields matters.
limitThe most documents to return.
skipHow many matches to jump over first.
page, pageSizePagination sugar: { page = 3, pageSize = 20 } means skip 40 and return at most 20. pageSize defaults to 20. Explicit limit and skip win if you pass both.
projectionWhich fields to include or leave out, like { name = 1, _id = 0 }.

Filters

A filter is a table of the fields a document must have: { name = "ada" }. For anything beyond exact equality, helper functions build the standard MongoDB operators for you.

HelperMatches when the field...
mongo.eq(v), mongo.ne(v)equals / does not equal the value
mongo.gt(v), mongo.gte(v), mongo.lt(v), mongo.lte(v)is greater / at least / less / at most
mongo.between(min, max)is between the two values, ends included
mongo.inList(values), mongo.notInList(values)is one of / none of the values
mongo.exists(flag?)exists (true, the default) or is missing (false)
mongo.regex(pattern, flags?)matches a regular expression, for example flags "i" for any capitalization
mongo.allIn(values)is an array containing all the values
mongo.size(n)is an array with exactly n items
mongo.notOf(operator)fails another operator test

Combine whole filters with mongo.anyOf({ ... }) (any may match), mongo.allOf({ ... }) (all must match), and mongo.noneOf({ ... }) (none may match).

src/main.luau
local adults = users:find(
    { age = mongo.gte(18), deletedAt = mongo.exists(false) },
    { sort = { age = -1 }, page = 1, pageSize = 25 }
)

local either = users:find(mongo.anyOf({
    { role = "admin" },
    { name = mongo.regex("^ada", "i") },
}))

users:updateOne({ _id = someId }, { ["$set"] = { active = true } })

Ids and how values convert

Every document gets an _id. mongo.objectId() makes a fresh id object; mongo.objectId(hex) rebuilds one from its 24 character hex text; mongo.isObjectId(value) checks for one. Id objects convert to real MongoDB ObjectIds when written and come back as the same kind of object when read. tostring(id) or id:hex() gives the hex text, and two ids compare equal with == when they hold the same value.

src/main.luau
--#include[mongo, datetime]

local id = users:insertOne({ name = "ada", joined = datetime.now() })
local doc = users:findOne({ _id = id })
print(doc.joined:format("%B %d, %Y"))
Empty tables

Luau cannot tell an empty array from an empty document, so an empty table becomes an empty document. If a field must be an empty array, mark the table with serde.array from the serde library — mongo honors the same marker.
