Including libraries

Lehua ships with its own libraries. They are opt in per file. To use one, add an include header at the top of the file. Only the libraries you include are set up and available to your code - and when building a program, Lehua can compile a runtime that only contains those libraries.

LibraryWhat it does
fsFiles, folders, and paths.
processArguments, environment, running programs, system info.
serdeEncode and decode JSON, YAML, and TOML.
cryptographyHashing, compression, base64, encryption, passwords, random.
datetimeDates and times as objects you can format and compare.
regexReal regular expressions alongside Luau's string patterns.
stdioTerminal input, output, prompts, and colors.
luauLoad and run Luau code built at runtime.
urlParse, build, and encode URLs and query strings.
semverParse and compare semantic version numbers.
archiveCreate and extract zip and tar archives.
sqliteA real SQL database in a single local file.
mongoTalk to MongoDB clusters, databases, and collections.
netHTTP, TCP, UDP, IPC, DNS, WebSocket, and FTP.
randomSeedable random number generator objects.
taskSpawn, defer, delay, and schedule work in coroutines.
src/main.luau
--#include[fs, process]

local text = fs.read("./data.txt")
print(process.platform)

All of the Luau standard libraries are always available. That means string, table, math, os, coroutine, bit32, utf8, buffer, and vector work with no include header. The include header is only for Lehua's own libraries.

Including everything

Use all to include every Lehua library at once. This is handy while you are trying things out. For a smaller build, list only the libraries you use.

src/main.luau
--#include[all]

fs.write("./log.txt", "hello")
print(process.platform)
Lehua is built in Rust on mlua and tokio. Made for people who love Luau.
